# 打包发布说明

## Logo 与图标

- Logo 源文件：`build/icon.svg`
- 自动切图脚本：`scripts/generate-icons.py`
- 生成产物：`build/icon.png`、`build/icon.ico`、`build/icon.icns`、`resources/icon.png`、`build/icons/icon-*.png`

重新生成图标：

```bash
npm run icons
```

## 本地构建

```bash
npm run build
```

构建内容会输出到 `out/`，不会生成安装包。

## 打包产物

```bash
npm run dist:win
```

本地打包脚本默认使用 `--publish never`，不会读取 Git 仓库信息或创建 GitHub Release。

Windows 会生成：

- NSIS 安装包：`release/<version>/MinIO Client-<version>-setup.exe`
- 免安装版：`release/<version>/MinIO Client-<version>-portable.exe`

macOS / Linux：

```bash
npm run dist:mac
npm run dist:linux
```

## 发布建议

- 发布前先执行 `npm run build` 确认类型检查和 Electron 构建通过。
- 如需发布到 GitHub Release，请先把 `package.json` 中的 `repository` / `homepage` 改成真实仓库地址，再使用专门的发布脚本。
- Windows 正式发布建议配置代码签名证书，避免 SmartScreen 风险提示。
- 当前 Windows 发布脚本：`npm run publish:win`。
